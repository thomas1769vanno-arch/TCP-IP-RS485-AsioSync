# Memory Bank — TCP-IP-RS485-AsioSync

## Come usare la Memory Bank

Leggere sempre i seguenti file in quest'ordine prima di lavorare al progetto:

1. `projectbrief.md` — scopo, architettura, stato attuale
2. `protocol.md` — dettagli del protocollo seriale attuale e futuro
3. `progress.md` — stato avanzamento lavori e prossimi step

---

## Regole

- La Memory Bank è la fonte di verità tra sessioni.
- Aggiornare `progress.md` ogni volta che si completa un task significativo.
- Se si introduce un nuovo protocollo o si modifica l'architettura, aggiornare `protocol.md`.
